# OCR MCP Server

A high-performance Go-based [MCP (Model Context Protocol)](https://modelcontextprotocol.io) server that enables **text-only LLMs** (DeepSeek, Kimi, GLM, Minimax, etc.) to understand images by converting them into structured text before the LLM receives the context.

The LLM never sees the image directly тАФ it only receives structured Markdown/JSON generated by the OCR pipeline.

The OCR service runs as a separate Python process with PaddleOCR loaded into memory, avoiding the 1тАУ3 second model loading latency on every request.

## Quick Start

### Prerequisites

- Go 1.25+
- Python 3.11+ (for the OCR service)
- Docker & Docker Compose (optional, for containerized deployment)

### Running with Docker Compose (recommended)

```bash
# Copy environment config
cp .env.example .env

# Build and start everything
docker compose up --build
```

The MCP server will be available at `http://localhost:7070/mcp` and the OCR service at `http://localhost:9090`.

### Running locally

**1. Start the OCR service (Python):**

```bash
cd services/ocr
pip install -r requirements.txt
python app.py
```

**2. Start the MCP server (Go):**

```bash
# Terminal 2
make build
make run
```

Or with hot reload:
```bash
make dev
```

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `SERVER_PORT` | `7070` | MCP server port |
| `OCR_SERVICE_URL` | `http://localhost` | OCR service URL |
| `OCR_SERVICE_PORT` | `9090` | OCR service port |
| `OCR_TIMEOUT` | `30s` | OCR request timeout |
| `OCR_MAX_RETRIES` | `3` | OCR retry count |
| `CACHE_TYPE` | `memory` | Cache backend (`memory` or `redis`) |
| `CACHE_TTL` | `1h` | Cache TTL |
| `CACHE_MAX_SIZE` | `1000` | Max cache entries |
| `WORKER_COUNT` | `4` | OCR worker pool size |
| `QUEUE_SIZE` | `100` | Worker pool queue depth |
| `MAX_IMAGE_SIZE_MB` | `20` | Max image upload size |
| `OUTPUT_FORMAT` | `markdown` | Output format (`markdown` or `json`) |
| `API_KEY` | `""` | API key for auth (empty = disabled) |
| `RATE_LIMIT_PER_MIN` | `60` | Rate limit per client |
| `LOG_LEVEL` | `info` | Log level |

## MCP Tools

### `read_image`

Extracts text from an image using OCR and returns structured results.

**Arguments:**

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `image_data` | string | Yes | Base64-encoded image data |
| `image_path` | string | No | File path (alternative to image_data) |
| `format` | string | No | Output format: `markdown` (default) or `json` |
| `preprocess` | string | No | Preprocessing: `auto`, `none`, `grayscale`, `threshold` |
| `api_key` | string | No | API key for auth (if configured on server) |
| `client_id` | string | No | Client identifier for rate limiting |

**Output (Markdown format):**

```markdown
# OCR Result

> **Provider:** PaddleOCR | **Time:** 42ms

## Extracted Text

Invoice #123
Amount тВ╣500

## Text Blocks

| # | Text | Confidence | Position |
|---|------|------------|----------|
| 1 | Invoice #123 | 99.0% | (10,10)-(200,30) |

## Tables

_No tables detected._

## Confidence

**Overall:** 98.0% (Very High)

**Range:** 95.0% тАУ 99.0%

## Layout

- **Elements detected:** 2
- **Height span:** 800px
- **Text density:** ~0.3 elements per 100px
```

## Monitoring

- **Health:** `http://localhost:7071/health`
- **Metrics:** `http://localhost:7071/metrics`

Metrics available via expvar:
- `requests_total` тАФ Total MCP tool requests
- `requests_active` тАФ Currently processing requests
- `cache_hits` / `cache_misses` тАФ Cache performance
- `ocr_calls` / `ocr_errors` тАФ OCR service calls
- `rate_limited` тАФ Requests denied by rate limiter
- `auth_failures` тАФ Failed authentication attempts
